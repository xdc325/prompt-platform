# Prompt 工程管理平台 — 项目总结

## 项目概述

一个将 LLM Prompt（提示词）视为工程资产进行全生命周期管理的平台，提供版本控制、回归测试、A/B 实验、效果对比、审核发布等功能。独立完成从设计到实现的全栈开发。

## 技术栈

| 层次 | 技术 | 用途 |
|------|------|------|
| 后端框架 | Python FastAPI + Uvicorn | 异步 REST API，自动生成 OpenAPI 文档 |
| ORM | SQLAlchemy 2.0（async） + asyncpg | 异步数据库操作，PostgreSQL 驱动 |
| 数据库 | PostgreSQL 16 | 主数据存储 |
| 缓存 / 队列 | Redis 7 + ARQ | 缓存 + 异步后台任务队列 |
| 前端 | Vue 3 + Vite | SPA 前端，响应式组件 |
| 反向代理 | Nginx | 静态资源服务 + API 反向代理 |
| 容器化 | Docker Compose | 6 服务编排（postgres, redis, backend, worker, frontend, nginx） |
| 认证 | JWT（python-jose） | access token + httpOnly refresh token |
| LLM 集成 | 多 Provider 抽象层 | 支持 OpenAI / DeepSeek / Claude，按模型名自动路由 |

## 核心功能

### 1. 项目与权限管理
- 创建 / 删除项目，Owner / Editor / Viewer 三级成员权限
- 项目级删除自动级联清理所有关联数据（prompts, versions, test suites, experiments）

### 2. 提示词版本控制
- 版本 CRUD、草稿 → 待审核 → 已发布状态流转
- 版本对比（逐行 diff，前端可视化展示增/删/改）
- 版本回滚（将已发布版本设为当前版本）
- 版本删除（带 FK 约束安全处理：清除 current_version_id 和 parent_version_id 引用）

### 3. 回归测试套件
- 创建测试套件（名称 + 多组输入/期望关键词）
- 选择版本和模型，异步后台执行（ARQ worker）
- 实时轮询测试结果，展示通过率和每个用例的输出
- 冲突保护（同一套件同时只能运行一个测试）
- 级联删除（删除套件时自动清理关联的 test_runs）

### 4. Playground 与版本对比
- 单版本即时代码执行（Playground），输入变量 `{input}` 模板替换
- 双版本对比（Compare）：同时执行两个版本，对比延迟、输出、逐词 diff

### 5. A/B 实验
- 创建实验（指定两个版本 + 流量分配比例）
- 查看实验汇总统计
- 停止实验

### 6. 多 LLM Provider 支持
- Provider 工厂模式：按模型名前缀自动选择 OpenAI / DeepSeek / Claude
- API Key 前置校验，避免无效请求卡死（60s 超时 → 即时 409 报错）
- 前端模型选择器支持 6 种模型

### 7. 用户认证
- 邮箱注册 + 登录
- JWT access token（30 分钟）+ httpOnly refresh token cookie
- 前端路由守卫 + 401 自动跳转登录页

## 项目规模

- **后端代码**：61 个 Python 文件，约 3,400 行
- **前端代码**：9 个源文件（3 视图 + 路由/API/组合式函数），约 1,100 行
- **API 端点**：35 个 REST 端点（含 SSE 流式端点）
- **数据库表**：9 张（users, projects, project_members, prompts, prompt_versions, test_suites, test_runs, experiments, experiment_results）
- **E2E 测试**：47 个测试用例，覆盖所有核心流程
- **Docker 服务**：6 个（postgres, redis, backend, worker, frontend, nginx）

## 架构设计

```
Nginx (:80)
  ├── /          → Vue 3 前端静态资源
  └── /api/v1/*  → FastAPI 后端 (:8000)
                     ├── api/        — 7 个路由模块
                     ├── services/   — 7 个业务服务（含权限混入）
                     ├── repositories/ — 9 个数据访问层（BaseRepository 模式）
                     ├── models/     — 9 个 SQLAlchemy ORM 模型
                     ├── providers/  — 4 个 LLM Provider（工厂模式）
                     ├── schemas/    — Pydantic 请求/响应验证
                     └── worker/     — ARQ 后台任务（回归测试执行）
```

### 分层架构

```
Router → Service → Repository → Model (SQLAlchemy)
              ↘ Provider (LLM 调用)
```

- **Router**：HTTP 层，解析请求、调用 Service、返回统一 `{success, data, error}` 格式
- **Service**：业务逻辑层，权限校验（PromptAccessMixin）、状态机、冲突检测
- **Repository**：数据访问层，封装 SQLAlchemy 查询，继承通用 BaseRepository
- **Provider**：LLM 调用抽象，支持 OpenAI / DeepSeek / Claude，统一 `ChatResult` 返回

## 解决的关键技术问题

### 1. 外键约束级联删除（3 处修复）

**问题**：直接删除带 FK 引用的记录导致 PostgreSQL 外键冲突。

**解决**：
- 版本删除前先清理 `prompts.current_version_id` 和 `prompt_versions.parent_version_id` 引用
- 项目删除实现完整级联链：prompts → versions → test_runs/experiments → suites → members
- 测试套件删除前先删除关联 test_runs

### 2. 后台任务队列未生效（2 处修复）

**问题**：测试套件运行后状态永远停留在 "running"，job 从未入队，结果从未写入数据库。

**根因**：
- `TestService` 构造函数接受 `arq` 参数但 API 层从未注入（始终为 None）
- Worker 任务执行完未调用 `session.commit()`，修改丢失

**解决**：
- 在 `deps.py` 添加 `get_arq()` 依赖注入，创建 ArqRedis 连接池
- Worker 任务末尾添加 `await session.commit()`
- 修复 datetime 时区冲突（`datetime.now(timezone.utc)` → `datetime.utcnow()` 匹配 `TIMESTAMP WITHOUT TIME ZONE` 列）

### 3. Worker 模型硬编码导致 DeepSeek 用户无法使用

**问题**：Worker 写死 `gpt-3.5-turbo`，用户只有 DeepSeek API Key 时测试必定失败。

**解决**：
- `TestRunRequest` schema 添加 `model` 字段（默认 `deepseek-chat`）
- ARQ job 参数传递 model，worker 动态选择 Provider
- 前端添加模型下拉选择器

### 4. API Key 缺失时请求卡死

**问题**：无 API Key 时，httpx 以 `Bearer None` 发请求，在外层超时（60s）后才返回，用户体验极差。

**解决**：在 Service 层添加 API Key 前置校验，对应模型前缀检查对应 Key 是否存在，不存在立即返回明确中文错误信息（如 "DeepSeek API Key 未配置"）。

### 5. Worker 输入渲染逻辑错误

**问题**：Worker 将测试用例的 `input`（字符串类型）当作 dict 遍历 `.items()`，导致字符级替换而非变量替换。

**解决**：改为 `version.content.replace("{input}", case_input)` 直接替换占位符。

### 6. 前端交互问题（3 处修复）

- 展开/收起按钮：阈值从 8 行调整为 12 行，CSS max-height 从 120px 调整为 200px
- 关闭对比按钮：从 float 布局改为 flexbox header，解决与内容的叠加问题
- 删除按钮缺失：为项目和版本添加删除按钮，带确认提示

## 设计原则实践

- **不变性**：所有数据操作返回新对象，不修改输入参数
- **KISS**：单体 FastAPI 应用，不做微服务拆分
- **DRY**：BaseRepository 抽象通用 CRUD，PromptAccessMixin 复用权限校验
- **YAGNI**：不做 Kubernetes、WebSocket、OAuth、RBAC 细粒度权限
- **防御性编程**：系统边界严格校验（Pydantic schema + API Key 前置检查）
- **错误处理**：统一 `{success, data, error}` 响应格式，异常 → HTTP 状态码映射
