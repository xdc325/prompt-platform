# Prompt 管理平台 — 完整功能测试手册

> 测试地址：`http://localhost`  
> 每个步骤按顺序执行，包含操作说明和**预期结果**，方便对照。

---

## 一、注册与登录

### 步骤 1：打开登录页
- **操作**：浏览器打开 `http://localhost`
- **预期**：自动跳转到登录页，看到「Prompt 管理平台」标题、邮箱/密码输入框、登录按钮、底部「没有账号？去注册」链接

### 步骤 2：切换到注册模式
- **操作**：点击「去注册」链接
- **预期**：密码框上方多出一个「显示名称」输入框，按钮文字变为「注册」，底部文字变为「已有账号？去登录」

### 步骤 3：注册新账号
- **操作**：邮箱填 `tester@demo.com`，显示名称填 `测试员`，密码填 `12345678`，点击「注册」
- **预期**：注册成功后自动跳转到项目列表页，顶部导航栏显示「项目列表」和邮箱 `tester@demo.com` 以及「退出登录」按钮

### 步骤 4：退出后重新登录
- **操作**：点击导航栏「退出登录」
- **预期**：跳回登录页
- **操作**：切换到登录模式，输入 `tester@demo.com` / `12345678`，点击「登录」
- **预期**：成功进入项目列表页

## 二、项目管理

### 步骤 5：查看空项目列表
- **操作**：登录后观察项目列表区域
- **预期**：显示「暂无项目，点击上方按钮创建第一个」

### 步骤 6：创建第一个项目
- **操作**：点击「+ 新建项目」按钮，项目名称填 `智能客服系统`，描述填 `客服AI提示词管理`，点击「创建」
- **预期**：右上角弹出绿色 Toast 提示「项目创建成功」，项目卡片出现在列表中

### 步骤 7：创建第二个项目
- **操作**：再创建一个项目，名称 `营销文案平台`，描述不填，点击「创建」
- **预期**：两个项目卡片都在列表中，第二个的描述显示「暂无描述」

## 三、提示词管理

### 步骤 8：选中项目查看提示词
- **操作**：点击 `智能客服系统` 项目卡片
- **预期**：卡片变为蓝色边框高亮，下方出现「智能客服系统 — 提示词」区域，显示「暂无提示词，点击上方按钮创建第一个」

### 步骤 9：创建提示词

- **预期**：Toast 提示「提示词创建成功」，卡片出现，显示名称和描述

### 步骤 10：再创建两个提示词
- **操作**：创建 `售后问题处理` 和 `投诉升级话术`，描述随意
- **预期**：三个提示词卡片都出现在列表中，每个显示更新时间和描述

## 四、版本管理（核心功能）

### 步骤 11：进入提示词详情页
- **操作**：点击 `售前咨询回复` 提示词卡片
- **预期**：跳转到详情页，顶部有「← 返回项目列表」按钮，显示提示词名称、描述，以及「版本管理」区域和「+ 新建版本」按钮

### 步骤 12：创建第一个草稿版本
- **操作**：点击「+ 新建版本」，在文本框中输入以下内容：
  ```
  你是{{company}}的智能客服，请用热情专业的语气回答客户问题。

  客户：{{customer_message}}

  回答要求：
  1. 先问候客户
  2. 针对问题给出具体答复
  3. 询问是否还有其他需要
  ```
  变更说明填 `初版客服话术模板`，点击「创建草稿」

- **预期**：Toast 提示「草稿创建成功」，版本卡片出现，显示：
  - v1
  - 状态标签「草稿」（灰色）
  - 变更说明标记（蓝色，鼠标悬停显示内容）
  - 内容预览（超过 8 行时折叠，显示「展开」按钮）
  - 操作按钮：提交审核、对比、测试

### 步骤 13：展开/折叠长内容
- **操作**：点击版本卡片的「展开」按钮
- **预期**：内容完全展开，按钮变为「收起」；再点一次恢复折叠

### 步骤 14：尝试直接发布草稿（非法操作）
- **预期**：草稿状态只有「提交审核」按钮，没有「发布」按钮。这验证了状态机正确——草稿不允许直接发布。

### 步骤 15：提交审核
- **操作**：点击「提交审核」按钮
- **预期**：Toast 提示「已提交审核」，版本状态变为「待审核」（黄色标签），按钮变为「发布」

### 步骤 16：发布版本
- **操作**：点击「发布」按钮
- **预期**：
  - Toast 提示「版本已发布」
  - v1 状态变为「已发布」（绿色标签）
  - 出现绿色「当前版本」徽章
  - 按钮只剩「对比」和「测试」（发布后不可删除）

### 步骤 17：验证返回后当前版本保持
- **操作**：点击「← 返回项目列表」，重新点击 `售前咨询回复` 进入
- **预期**：v1 仍然是「已发布」状态，绿色「当前版本」徽章还在

### 步骤 18：创建 v2 并发布（验证自动归档）
- **操作**：
  1. 点击「+ 新建版本」
  2. 内容改为：
     ```
     你是{{company}}的高级客服专家，请用{{tone}}的语气回答客户问题。

     客户问题：{{customer_message}}

     回复要求：
     1. 先自我介绍并问候
     2. 针对问题给出专业答复
     3. 提供进一步帮助的选项
     4. 以感谢语结尾
     ```
  3. 变更说明填 `增加语气变量和结构化回复`
  4. 创建草稿 → 提交审核 → 发布

- **预期**：
  - v2 变为「已发布」并带上「当前版本」徽章
  - v1 自动变为「已归档」（灰色标签），「当前版本」徽章消失
  - v1 出现红色「删除」按钮（因为归档状态可删除）

## 五、版本对比

### 步骤 19：对比两个版本
- **操作**：
  1. 先点击 v1 的「对比」按钮
  2. 页面顶部出现蓝色提示框「请选择第二个版本进行对比」
  3. 再点击 v2 的「对比」按钮

- **预期**：
  - 底部出现「版本对比」区域，标题显示 `v1 → v2`
  - 列出变更详情，包含多个变更条目：
    - `removed`（红色）- 被删除的行
    - `added`（绿色）- 新增的行
    - `replaced`（黄色）- 被修改的行
  - 底部有摘要文字描述变更数量

### 步骤 20：关闭对比
- **操作**：点击对比区域的「关闭」按钮
- **预期**：对比区域消失

## 六、即时测试（Playground / DeepSeek）

### 步骤 21：打开即时测试
- **操作**：点击 v2 的「测试」按钮
- **预期**：页面底部出现「即时测试 — v2」区域，包含：
  - 文本输入框
  - 模型下拉菜单（OpenAI 分组 + DeepSeek 分组，默认 GPT-3.5 Turbo）
  - 「运行」和「关闭」按钮

### 步骤 22：用 DeepSeek 测试
- **操作**：
  1. 模型选择 `DeepSeek Chat`
  2. 输入框填 `你好，我想退货`
  3. 点击「运行」

- **预期**：等待几秒后，出现「输出结果」区域，显示 DeepSeek 返回的回复内容。回复应该：
  - 自称是客服专家
  - 针对退货问题给出答复
  - 结构符合模板要求（问候 + 答复 + 帮助 + 感谢）

### 步骤 23：关闭并重新测试
- **操作**：关闭即时测试，再点一次「测试」，这次模型选 `GPT-4`，输入相同内容
- **预期**：如果未配置 OpenAI Key，会报错提示。如果有 Key，会正常返回

## 八、版本删除

### 步骤 26：删除归档版本
- **操作**：点击 v1 的红色「删除」按钮
- **预期**：浏览器弹出确认对话框「确定要删除这个版本吗？此操作不可撤销。」
- **操作**：点击「确定」
- **预期**：Toast 提示「版本已删除」，v1 从列表中消失

### 步骤 27：删除当前版本 v2
- **操作**：v2 现在是「已发布」状态，不应该出现删除按钮
- **预期**：v2 只有「对比」和「测试」按钮，没有删除按钮。这验证了已发布版本不能直接删除的限制。

## 九、测试套件

### 步骤 28：通过 API 创建测试套件（前端暂未实现此 UI）
当前测试套件功能通过 API 使用，在终端执行：

```bash
# 获取 token
TOKEN=$(curl -s -X POST http://localhost/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"tester@demo.com","password":"12345678"}' | python3 -c "import sys,json; print(json.load(sys.stdin)['data']['access_token'])")

# 创建测试套件
curl -s -X POST "http://localhost/api/v1/prompts/{prompt_id}/test-suites" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $TOKEN" \
  -d '{
    "name": "回归测试集",
    "test_cases": [
      {"input": "{\"customer_message\": \"我要退货\", \"company\": \"测试商城\", \"tone\": \"热情\"}", "expected": "退货"},
      {"input": "{\"customer_message\": \"物流太慢了\", \"company\": \"测试商城\", \"tone\": \"耐心\"}", "expected": "物流"}
    ]
  }'

# 运行测试
curl -s -X POST "http://localhost/api/v1/prompts/{prompt_id}/test-suites/{suite_id}/run" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $TOKEN" \
  -d '{"version_id": "{version_id}"}'

# 查看结果（用返回的 run_id）
curl -s "http://localhost/api/v1/test-runs/{run_id}" \
  -H "Authorization: Bearer $TOKEN"
```
